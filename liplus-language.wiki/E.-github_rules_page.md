# GitHub Operational Rules

### Common Format for Issue / Commit / Pull Request

- **Title**
  - English
  - ASCII only
  - Single line

- **Body**
  - Japanese
  - **Multi-language support may be added in the future**

### Additional Requirements for Commit / Pull Request

- **Issue number is mandatory**
  - Must be included in the body
  - Example: Refs #123

### Additional Requirements for Pull Request

- **Body must include a short summary**
  - 2 –3 lines
  - Summarize the change only
  - Detailed explanations should remain in Issues or Commits

---

These rules are expressly operational.

If a change does not conform to these rules, it must be corrected before merging.
